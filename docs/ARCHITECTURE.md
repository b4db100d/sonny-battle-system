# Architecture

Static Protocol is a Godot 4.4 turn-based RPG built mobile-first
(landscape, touch). Two decisions shape everything else:

1. **The combat engine is pure logic.** Everything under `src/` uses only
   `RefCounted`/`Resource` — no Nodes, no scene tree, no autoload access.
   The entire game can be simulated headlessly (see
   `tests/test_campaign_sim.gd`, which auto-plays the whole campaign in CI).
2. **All content is data.** Abilities, statuses, enemies, items, stages,
   zones, and skill trees are custom `Resource`s (`.tres` files under
   `data/`), generated from the declarative spec in `tools/gen_content.py`.
   Saves store only string ids; the `Db` autoload rehydrates them.

## Layers

```
data/*.tres            content (generated by tools/gen_content.py)
src/data/*.gd          Resource class definitions (AbilityData, EnemyData, ...)
src/combat/*.gd        pure battle engine
src/progression/*.gd   pure stat/XP/tree/loot/equipment math
autoload/*.gd          singletons: EventBus, Db, GameState, SaveManager,
                       AudioManager (stub), SceneRouter
scenes/                view layer (procedural UI; .tscn files are minimal)
tests/                 headless test suite + campaign simulator
```

## Combat engine (`src/combat/`)

`BattleState` owns a battle. The view (or a simulation) drives it with
three calls — `start()`, `act(ability_id, target_index)`, `act_pass()` —
each returning an ordered **event list** (`{"type": "damage", "target": 2,
"amount": 41.3, "crit": false}` …). The battle scene replays those events
as animations; the simulator just counts them. `BattleState.phase` tells
the caller what's next: `AWAIT_INPUT`, `VICTORY`, or `DEFEAT`.

- `TurnQueue` — per-round order by derived Speed, seeded tie-break;
  rebuilt every round so speed buffs/debuffs matter.
- `AbilityResolver` — damage/heal/crit/dodge/focus math. ±10% variance,
  crits ×1.6, shields absorb first.
- `StatusEngine` — application (REFRESH/STACK/IGNORE), DoT/HoT ticks with
  caster-power snapshots, expiry, shields, dispels.
- `EnemyAi` — scores every legal (ability, target) pair (profile
  multipliers, missing-HP heal weighting, kill-shot bonus, stun-the-hitter,
  anti-status-overwrite), then picks among the top 3 with a temperature:
  difficulty Easy/Normal/Hard = noisy/balanced/near-optimal play. The same
  AI drives the player's side in simulations.
- `RngService` — seedable wrapper; same seed ⇒ identical battle.

Turn lifecycle: cooldowns tick and focus regenerates at the holder's turn
start; `TURN_START` statuses tick (stun is checked after, skipping the
action); `TURN_END` statuses tick and durations expire at turn end.
Multi-wave stages spawn the next wave when the field is cleared.

## Stats and progression

Primaries: **Strength** (physical power), **Instinct** (psy power, heals,
focus regen), **Speed** (turn order, crit, dodge), **Vitality** (HP).
`StatBlock.derive()` is the single formula source. Statuses apply
multiplier deltas to primaries; equipment adds flat primaries plus flat
derived bonuses (`max_hp`, `crit_chance`, …).

Leveling: `xp_to_next = 60 · level^1.55`, cap 24; +2 stat / +1 skill point
per level; respec is free. Three 6-ability trees (Havoc/Surge/Bastion)
gated by level and points-in-tree (0/1/3/5 per tier); at most 6 abilities
equipped. Allies (Vela, Brick) have fixed kits and level with the player.

## Game flow

`MainMenu → ZoneMap → [DialogueBox] → Battle → results → ZoneMap`, with
`CharacterSheet` reachable from the map. `SceneRouter` stashes a payload
dictionary across `change_scene_to_file`. Saves are versioned JSON in
`user://saves/slot_N.json` (autosave after battles); `GameState.from_dict`
normalizes JSON floats back to ints.

## View layer conventions

`.tscn` files contain a root node + script only; all children are built in
`_ready` (hand-editing scene text is the most error-prone part of working
without the editor, so we minimize it). UI is anchor-based for any aspect
ratio; combatants are `Area2D`s with generous tap hitboxes; all animation
is procedural tweens.

## Testing

`tools/validate.sh` = import + full suite via the custom runner
(`tests/run_tests.gd`, no addons). Coverage: stat/XP math, resolver
determinism, status stacking, battle flow (taunt/stun/waves/defeat), save
round-trip, content integrity (every `.tres` + cross-references), scene
instantiation, all-scripts compile, fixture auto-battles, and the
campaign simulator with its balance report.

## Deliberate cuts (future work)

- No shops/economy beyond credits accrual; loot comes from battles.
- Allies are not customizable (fixed kits, auto stat spread).
- Audio is a no-op stub (`AudioManager`); hook real SFX/music there.
- Art is programmer-placeholder (colored polygons); swap `CombatantView`
  body-building for sprites when art exists.
- Single save-slot UI (the SaveManager supports 3 slots already).
